I know, it‚Äôs pretty funky syntax, especially when you‚Äôve got lots of variables in your css (and even more so when you‚Äôre using variables as fallbacks to variables), but at least the usefulness far outweighs this.

You‚Äôll notice in the first example that the variable has been defined inside the block for the html element. This is important, because, like other programming languages, variables in CSS also have scopes. Variables need to be defined in a certain scope, and then they can be used anywhere within that scope. In the example above, the scope is html, and so --my-variable can be used within that element and all its children as well.
